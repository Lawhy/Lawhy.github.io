Chat Template Correctness

Apply the chat template once for initial context, then incrementally append environment-rendered messages.

1. Initial context via apply_chat_template

<|im_start|>system
You are a research assistant.
# Tools: search, scrape<|im_end|>
<|im_start|>user
Summarize DeepSeek-V3's key innovations<|im_end|>
<|im_start|>assistant

↓ model generates

2. Model-generated assistant segment

<think>I should find the report</think>
<tool_call>{"name":"search","arguments":{...}}</tool_call><|im_end|>

↓ environment gives new prompt

3. Tool result + next assistant prefix

\n ← msg_sep
<|im_start|>user
<tool_response>[1] "DeepSeek-V3 Report — arxiv.org"</tool_response><|im_end|>
<|im_start|>assistant

Problem: how to tokenize intermediate prompts while preserving TITO?

Bad options:

  • Re-template full trajectory — may alter model-generated tokens.
  • Template one message alone — may inject BOS (e.g., GLM's [gMASK]<sop>), miss separators, or fail entirely (some templates don't support it).

Solution: the fake prefix trick.

Fake Prefix Trick
1. fake_msgs ← [fake_sys, fake_user]
2. full_prompt ← chat_template(fake_msgs + [new_msg])
3. prefix_prompt ← chat_template(fake_msgs)
4. prompt ← msg_sep + full_prompt[len(prefix_prompt):]
5. new_ids ← encode(prompt)

msg_sep: separator between <|im_end|> and next <|im_start|>.

Correctness matters!

(1) Missing msg_sep → token misalignment.  (2) Skipping incremental rendering → breaks TITO — unless intentional (e.g., context summarization). (3) Chat templates are essentially for serving — for training correctness, they may still need modification or monkey-patching.
