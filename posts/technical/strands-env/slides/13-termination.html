Termination Reasons

Before a reward, the environment catches what reason ended the rollout. Turning each reason into how we treat a rollout is an algorithm choice and a lever for the model's behavior. A few examples:

throttling timeout
→
retry or drop — the env's fault, not about the policy
context_window_overflow
→
length penalty — if the model is too verbose and bloats the context in just a few turns
max_tool_iterations
→
negative sample — an unexpectedly long rollout in an easy task; truncated and penalized, even if it might've reached the answer
task_complete
→
grade the output — it finished on its own; hand to the reward function
