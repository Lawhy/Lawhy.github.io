Async ≠ Parallelism

Agent loop is async by default — a rollout is mostly waiting (model/tool calls) in one event loop. But Python's async is one thread, bounded by its GIL. 

@tool
async def read_page(url):
    html = await fetch_html(url)      # awaits → releases the loop
    markdown = html_to_markdown(html) # no await → holds the loop
    return markdown
× So html_to_markdown is CPU-side work with no await: each call is cheap, but blocking sections pile up across thousands of concurrent rollouts.
