Tool Parsing

Tool parsers on inference servers (e.g., SGLang/vLLM) silently fix what the model got wrong — for RL, they must fail loud. Examples:

{"name": "search",
 "arguments": {
   "query": "deep
→
{"name": "search",
 "arguments": {
   "query": "deep"
 }}
Parsers may silently patch the format, the broken call lands as “success” like nothing happened. 
<think>let me try
  <tool_call>search("X")</tool_call>
</think>
<tool_call>search("Y")</tool_call>
→
[#1] search("X")
[#2] search("Y")
Draft tool calls inside the think block got executed too.
