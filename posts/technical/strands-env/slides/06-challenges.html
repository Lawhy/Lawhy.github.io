But to make it trainable...

… the agent's rollout has to be both correct and efficient — before RL algorithm optimizations.

Correctness
  • Tokens — no retokenization drift
  • Tool calls — strict parsing, no fallbacks
  • Terminations — classified to shape reward
  • Train–inference match — kernel-level; not elaborated in this presentation
Efficiency
  • Async ≠ parallelism — overlaps I/O, not CPU
  • Bottlenecks — CPU, rate-limits, slow tools
  • Per-env tuning — no single solution
  • Fully-async rollouts — off-policy data trade-off for speed
